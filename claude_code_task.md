# 电协 Hub —— 用户头像功能需求

## 背景

Next.js (App Router, TypeScript) + Prisma + PostgreSQL + Auth.js + Tailwind + Supabase Storage 的电协内部网站（EEA_Hub），暗色主题，已上线。已有 Supabase Storage（bucket: eea-hub）用于帖子配图和作业文件，可复用。论坛/文档有匿名机制（匿名内容对外显示"匿名楼主/匿名用户A/B/C"，绝不暴露真实身份）。

本次：增加用户头像功能。

配色板：主背景 #212733、卡片 #2A3140、主文字 #E8ECF1、主交互 #4FD1C5、次级 #7CCACD/#6D96B4、成功 #6DAB70、暖金 #C4AC60、边框 #3A4250。

## 一、头像上传与存储

1. **上传入口**：在个人主页（/dashboard 或个人资料区）增加"上传/更换头像"功能
2. **存储**：头像图片上传到 Supabase Storage（复用现有 bucket 和上传逻辑），拿到 URL
3. **数据库**：User 表增加 `avatarUrl String?` 字段，保存头像 URL（生成 migration）
4. **上传体验**：
   - 支持常见图片格式（jpg/png/webp）
   - 建议限制文件大小（如 2MB 以内）
   - 上传后个人主页即时显示新头像
   - 上传出错给中文友好提示，不要 500

## 二、两种内置头像（放在 public/ 目录）

项目会放入两张内置头像图片（用户会提供）：
- `public/default-avatar.png` —— **默认头像**：实名用户未设置头像时显示
- `public/anonymous-avatar.png` —— **匿名头像**：所有匿名内容显示

代码中通过常量引用这两个路径，方便以后替换图片。

## 三、头像显示规则（重点：匿名安全）

在**所有显示作者的地方**显示头像，包括：
- 论坛帖子（作者）
- 论坛回复（作者）
- 文档（作者）
- 文档评论（如果已实现）
- 个人主页

**显示逻辑（务必严格遵守，涉及匿名安全）：**

1. **实名内容**：
   - 作者设置了头像 → 显示其 avatarUrl
   - 作者未设置头像 → 显示默认头像 `default-avatar.png`

2. **匿名内容**（匿名帖、匿名回复、匿名评论等）：
   - **一律显示匿名头像 `anonymous-avatar.png`**
   - **绝对不能**显示作者的真实头像（否则通过头像就泄露了匿名者身份，破坏匿名机制）
   - 即使该匿名作者设置了头像，匿名内容也只显示匿名头像

3. **个人主页**：显示当前登录用户自己的头像（设置了显示真实头像，没设置显示默认头像）

**安全核心**：任何对外公开的匿名内容，头像只能是匿名头像，不得因头像功能泄露匿名者身份。这一点必须在渲染层严格保证——渲染匿名内容时，不要把真实 avatarUrl 传到前端。

## 四、头像组件

建议封装一个统一的 Avatar 组件，处理：
- 圆形裁剪显示
- 传入 avatarUrl 显示图片，为空显示默认头像
- 匿名场景传入"匿名"标志，显示匿名头像
- 合适的尺寸变体（如列表里小尺寸、个人主页大尺寸）
- 符合暗色主题

各处显示作者的地方统一用这个组件，保证一致性。

## 五、技术要求

- 头像上传走 server action，复用现有 Supabase 上传逻辑
- User 表加 avatarUrl 字段，生成 migration
- 匿名渲染逻辑严格：匿名内容不传真实 avatarUrl 到前端
- 出错给中文提示，不要 500
- 改完跑 npm run prisma:generate、npm run lint、npm run build，三项全部通过

## 六、验收要点（自测）

- 能在个人主页上传/更换头像，上传后立即显示
- 实名帖子/回复/文档显示作者头像；未设置头像的显示默认头像
- 匿名帖子/回复/评论一律显示匿名头像，不显示真实头像（重点验证：给一个设置了头像的账号发匿名帖，确认匿名帖显示的是匿名头像而非他的真实头像）
- 各处头像样式统一、圆形、符合暗色主题

## 七、本次不做

- 头像裁剪编辑器（直接上传即可，不做裁剪交互）
