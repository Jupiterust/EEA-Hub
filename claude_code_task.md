# 电协 Hub —— 搜索增强需求（关键词高亮 + 按作者筛选）

## 背景

Next.js (App Router, TypeScript) + Prisma + Tailwind 的电协内部网站（EEA_Hub），暗色主题，已上线。

现有搜索功能：
- 全站搜索（/search）：搜文档+帖子标题/正文，结果只显示标题，无高亮
- 文档列表（/docs）：按标题/正文搜索，有搜索框，无高亮无作者筛选
- 论坛列表（/forum）：按标题/正文/标签搜索，有搜索框，无高亮无作者筛选

本次两块：1）搜索结果关键词高亮；2）按作者筛选。

配色：主背景 #212733、卡片 #2A3140、主文字 #E8ECF1、主交互 #4FD1C5、次级 #6D96B4、暖金 #C4AC60、边框 #3A4250。

---

## 第一块：搜索结果关键词高亮

在以下三个页面，搜索有结果时，把匹配到关键词的部分高亮显示：

### 范围
- `/search`（全站搜索）：结果卡片里的标题高亮
- `/docs`（文档列表）：文档卡片的标题高亮
- `/forum`（论坛列表）：帖子卡片的标题高亮

### 实现方式
- 用简单的字符串分割 + `<mark>` 标签实现，不引入复杂依赖
- 把标题按关键词分割，匹配部分用 `<mark>` 包裹，其余部分正常渲染
- 大小写不敏感（和现有搜索查询一致）
- `<mark>` 样式：符合暗色主题，建议用主交互色 `#4FD1C5` 的浅色背景（如 `bg-primary/20 text-primary rounded px-0.5`），不要用默认黄色（和暗色主题不搭）
- 封装成一个 `HighlightText` 工具组件，在三个页面复用

## 第二块：按作者筛选

在文档列表、论坛列表、全站搜索加上"按成员（作者）筛选"功能。

### 文档列表（/docs）
- 现有：按标题/正文的文字搜索框
- 新增：旁边加一个"按作者"筛选下拉或输入框
- 实现：URL 参数加 `author`（如 `/docs?author=李睿` 或 `/docs?authorId=xxx`）
- 查询：筛选 author.realName 或 author.username 包含输入值的文档

### 论坛列表（/forum）
- 现有：按标题/正文/标签搜索
- 新增：加一个"按作者"筛选
- 注意匿名帖：匿名帖不显示作者，按作者筛选时**不应该**把匿名帖匹配出来（即使数据库里有 authorId）——保护匿名性
- 实现：筛选 `isAnonymous: false` 且 author.realName/username 包含输入值的帖子

### 全站搜索（/search）
- 现有：只有关键词搜索框
- 新增：加一个"按作者"输入框，和关键词搜索组合使用
- 按作者搜时，文档和帖子都按作者筛选（帖子同样排除匿名帖）

### 作者筛选的 UI
- 简单的文字输入框（输入成员名字/用户名），和现有搜索框并排或在下方
- 通过 URL 参数传递（`?author=xxx`），和现有的 `?q=xxx` 组合使用
- 筛选和关键词搜索可以同时使用（比如：搜索"STM32" + 作者"李睿"）
- 有作者筛选时，保持现有的排序功能（论坛的最新/最多回复/最多点赞）

---

## 匿名保护（重要）

论坛和全站搜索的"按作者筛选"，必须排除匿名帖：
- 只匹配 `isAnonymous: false` 的帖子
- 不能因为按作者搜索暴露"某个匿名帖其实是某人发的"

---

## 技术要求

- 关键词高亮：封装 HighlightText 组件，三处复用
- 按作者筛选：通过 URL 参数实现，服务端查询，无需新增 API
- 不改数据库 schema，不加新字段
- 改完跑 npm run lint、npm run build，两项通过

## 验收要点

- 搜索"STM32"→ 结果里"STM32"高亮显示（青色背景）
- 文档列表/论坛列表/全站搜索都有高亮
- 按作者输入"李睿"→ 只显示李睿发的内容
- 关键词+作者同时筛选正常工作
- 论坛按作者筛选不暴露匿名帖（匿名帖不出现在结果里）
- 排序（最新/最多回复/最多点赞）和作者筛选可以同时使用

## 本次不做

- 正文内容的高亮（只做标题高亮，正文高亮复杂且通常不显示在列表里）
- 按部门/小组筛选（文档列表已有 division 筛选）
- 模糊搜索/拼音搜索
