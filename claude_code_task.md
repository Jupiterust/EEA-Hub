# 电协 Hub —— 论坛楼中楼（嵌套回复）需求

## 背景

Next.js (App Router, TypeScript) + Prisma + PostgreSQL + Auth.js + Tailwind + Supabase 的电协内部网站（EEA_Hub），暗色主题，已上线。

论坛现有：帖子（ForumPost）、回复（ForumReply）。回复机制：实名/匿名（帖内编号"匿名用户A/B/C"，公开页不暴露真实身份，数据库保留 authorId 追溯）、点赞（ReplyLike）、采纳最佳答案、编辑/删除（作者改自己的、管理员删任何的，服务端校验）、通知（被回复/被点赞/被采纳会通知）。

**当前问题**：回复之间不能互相回复，只能对帖子发平级回复，无法形成针对某条回复的讨论。

**本次目标**：增加"楼中楼"——允许回复某条具体回复，形成 1 层嵌套的讨论。

配色板：主背景 #212733、卡片 #2A3140、主文字 #E8ECF1、主交互 #4FD1C5、次级 #7CCACD/#6D96B4、成功 #6DAB70、暖金 #C4AC60、危险 danger、边框 #3A4250。

---

## 一、嵌套结构：只嵌套 1 层 + @引用

采用"1 层嵌套"模型（类似贴吧/微博，手机友好）：

- **主回复**（直接回复帖子的回复）作为顶层楼层
- **子回复**（回复某条回复的回复）平铺挂在它所属的**主回复**下面，缩进一层显示
- **子回复不再继续往下嵌套**：即使 C 回复的是 B（而 B 是 A 的子回复），C 仍然平铺在主回复 A 这一层下面，通过 **@被回复者** 来标明回复对象

示例显示效果：
```
回复A（主楼层）
   ├ B：@A 你这个说法我有疑问…
   ├ C：@B 我同意B的观点…       ← C回复B，但仍平铺在A层下，用@B标明
回复D（另一主楼层）
   ├ E：@D …
```

实现要点：
- ForumReply 增加字段：`parentId String?`（指向所属主回复的 id；主回复的 parentId 为空）和 `replyToId String?`（指向"@的那条回复"的 id，用于显示 @对象，可选）
- 或采用你认为更合理的结构，但核心是：**只缩进 1 层**，子回复用 @ 标明回复的具体对象
- 计算"@谁"时显示对方的显示名（实名显示名字，匿名显示"匿名用户X"）

---

## 二、交互

1. **每条回复（主回复和子回复）都有"回复"按钮**，点击后出现回复输入框
2. 回复主回复 → 成为它的子回复
3. 回复某条子回复 → 也归到同一个主回复下（1层），并 @那条子回复的作者
4. 回复框支持匿名勾选（同现有回复）
5. 发表后刷新显示，新回复出现在对应主回复的子回复列表里

---

## 三、与现有机制的配合（重要，全部要兼容）

楼中楼的子回复，必须和现有回复一样支持：

1. **匿名**：
   - 子回复可匿名发表，匿名时显示"匿名用户X"（沿用帖内编号机制，编号在整个帖子范围内一致）
   - @的对象如果是匿名的，显示 @匿名用户X，**绝不暴露真实身份**
   - 公开页对所有人匿名，作者本人可在个人主页管理自己的匿名子回复

2. **点赞**：子回复也能点赞/取消（复用 ReplyLike 机制）

3. **编辑/删除**：
   - 作者本人可编辑/删除自己的子回复，管理员可删除任何
   - 服务端严格校验
   - 删除有二次确认
   - 删除一条有子回复的主回复时，合理处理其下的子回复（建议：一并删除，或标记"该回复已删除"但保留子回复，由你选更合理的）

4. **通知**：
   - B 回复了 A 的回复 → 通知 A（"有人/某人回复了你的回复"）
   - 匿名回复时通知显示"有匿名用户回复了你的回复"，不暴露身份
   - 自己回复自己不通知

5. **采纳最佳答案**：楼主采纳仍只针对主回复（子回复是否可被采纳由你判断，建议只主回复可被采纳，保持简单）

## 四、显示细节

- 主回复正常显示；子回复缩进一层（视觉上有层级感，但只缩进这一层）
- 子回复显示"@被回复者"前缀（实名显示名字，匿名显示"匿名用户X"）
- 子回复同样显示作者头像（匿名用匿名头像）、点赞数、编辑/删除入口、时间
- 子回复数量多时，可考虑"展开更多回复"的折叠（可选，不强制）
- 手机端：缩进不要过深导致内容被挤压，保证可读

---

## 五、技术要求

- ForumReply 加字段（parentId 等），生成 migration
- 所有写操作走 server action，服务端严格校验权限
- 匿名机制严格：公开页不暴露任何匿名者身份，@对象也不暴露
- 不要改坏现有的回复、点赞、采纳、通知、编辑删除功能（回归测试）
- 出错给中文提示，不要 500
- 改完跑 npm run prisma:generate、npm run lint、npm run build，三项全部通过

## 六、验收要点（自测）

- 能回复某条回复，子回复正确挂在主回复下，只缩进 1 层
- 子回复显示 @被回复者
- 子回复支持匿名（@匿名对象不泄露身份）、点赞、编辑删除
- B回复A的回复 → A收到通知；匿名回复显示"匿名用户"不泄露
- 现有功能没被改坏：普通的对帖子回复、点赞、采纳最佳答案、通知都正常
- 手机端嵌套显示不挤压、可读
- 删除主回复时子回复处理合理，不报错

## 七、本次不做

- 通知的删除/清空（下一批）
- 无限层级嵌套（只做 1 层）
- 子回复被采纳为最佳答案（仅主回复可采纳）
