# 电协 Hub —— 头像上传裁剪功能需求

## 背景

Next.js (App Router, TypeScript) + Tailwind + Supabase Storage 的电协内部网站（EEA_Hub），暗色主题，已上线。

头像上传已实现：个人主页头像角标点击 → 选文件 → 直接上传到 Supabase。但目前没有裁剪，整张图片直接上传，如果用户上传全身照或非正方形图片，头像显示会变形或显示效果差。

本次：在上传流程中加入**正方形裁剪步骤**，让用户自己选择头像显示区域。

---

## 功能流程

1. 用户点击头像右下角的相机角标
2. 选择图片文件（jpg/png/webp，≤2MB）
3. **弹出裁剪弹窗**：
   - 显示所选图片
   - 提供一个**正方形（1:1）裁剪框**，用户可拖动位置、缩放大小
   - 两个按钮："确认裁剪" 和 "取消"
4. 用户点"确认裁剪" → 把裁剪后的图片区域导出为图片数据
5. 裁剪后的图片上传到 Supabase Storage（复用现有 uploadAvatarAction）
6. 头像更新显示

---

## 技术实现

- 使用 `react-image-crop` 库实现裁剪交互（轻量、无额外依赖）
  - 安装：`npm install react-image-crop`
  - 固定裁剪比例 `aspect={1}`（正方形）
- 裁剪后用 canvas 把选区导出为 Blob，再上传
- 裁剪弹窗是一个模态框（Modal），样式符合暗色主题（深色背景、圆角、遮罩层）
- 修改现有的 `AvatarUpload` 客户端组件（`src/components/avatar-upload.tsx`）：
  - 文件选择后不再直接提交，改为打开裁剪弹窗
  - 裁剪确认后再调用上传 action
- 取消裁剪 → 关闭弹窗，不上传，头像不变

---

## UI 细节

- 裁剪弹窗：
  - 全屏遮罩（`bg-black/60`），弹窗居中
  - 弹窗背景用卡片色（`#2A3140`），圆角，适当 padding
  - 裁剪框用主交互色 `#4FD1C5` 边框
  - 底部两个按钮：「取消」（次级按钮）和「确认裁剪」（主色按钮）
- 裁剪框固定 1:1 比例，用户可以：
  - 拖动裁剪框位置
  - 拖动边角缩放大小
- 上传中显示 loading 状态（复用现有 pendingText 机制）

## 技术要求

- 安装 `react-image-crop`
- 只改 `src/components/avatar-upload.tsx`，不改其他组件和页面
- 上传逻辑复用现有 `uploadAvatarAction`（只是上传的数据从原图改为裁剪后的 Blob）
- 改完跑 npm run lint、npm run build，两项通过

## 验收要点

- 点头像角标 → 选图 → 弹出裁剪弹窗
- 裁剪框是正方形，可拖动位置、缩放
- 确认裁剪 → 上传裁剪后的区域 → 头像更新
- 取消 → 弹窗关闭，头像不变
- 上传非正方形图片（如全身照）→ 裁剪后头像显示正常，不变形
- 暗色主题弹窗样式正常

## 本次不做

- 旋转/翻转图片
- 自由比例裁剪（只做正方形）
- 裁剪后压缩图片大小（直接上传裁剪区域即可）
