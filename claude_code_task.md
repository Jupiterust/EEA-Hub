# 电协 Hub —— 文档评论楼中楼（嵌套回复）需求

## 背景

Next.js (App Router, TypeScript) + Prisma + PostgreSQL + Auth.js + Tailwind + Supabase 的电协内部网站（EEA_Hub），暗色主题，已上线。

**论坛刚刚完成了楼中楼功能**，本次要把同样的机制完整复刻到**文档评论**上。

论坛楼中楼的实现（作为参考，文档评论照此复刻）：
- ForumReply 有 parentId（指向所属主回复，1层拍平：回复子回复时 parentId 仍指向顶层主回复）、replyToId（指向被@的具体回复，用于显示@和跳转）、isDeleted（软删除标记）
- 1层嵌套：子回复平铺在主回复下，缩进一层，用 @被回复者 标明回应对象
- 软删除：删除一条被@过或有子回复的回复时，标记 isDeleted=true 显示"[该回复已删除]"占位（不暴露作者），保留其下讨论；无人回复无人@的回复才物理删除
- @可点击跳转：点 @某人 平滑滚动+高亮到被@的评论
- 匿名、点赞、编辑删除全部兼容

文档评论现有：DocComment（docId、authorId、content、isAnonymous）、DocCommentLike（点赞）、匿名帖内编号、编辑删除。

配色板：主背景 #212733、卡片 #2A3140、主文字 #E8ECF1、主交互 #4FD1C5、次级 #7CCACD/#6D96B4、成功 #6DAB70、暖金 #C4AC60、危险 danger、边框 #3A4250。

## 目标：文档评论支持楼中楼，完整复刻论坛实现

### 1. 数据库
DocComment 增加字段（对标 ForumReply）：
- `parentId String?` —— 指向所属主评论（1层拍平：回复子评论时 parentId 仍指向顶层主评论）
- `replyToId String?` —— 指向被@的具体评论，用于显示@和跳转
- `isDeleted Boolean @default(false)` —— 软删除标记
- 生成 migration

### 2. 嵌套结构（1层 + @引用）
- 主评论（直接评论文档）作为顶层
- 子评论（回复某条评论）平铺挂在所属主评论下，缩进一层
- 回复子评论时，新评论的 parentId 仍指向顶层主评论（拍平到1层），replyToId 指向实际被@的评论
- 子评论显示 @被回复者（实名显示名字，匿名显示"匿名用户X"，不暴露身份）

### 3. 交互
- 每条评论（主+子）都有"回复"按钮，点击展开内联回复框（带匿名勾选）
- 回复主评论 → 成为其子评论
- 回复子评论 → 归到同一主评论下（1层），@那条子评论作者
- 发表后刷新显示

### 4. 软删除占位
- 删除一条评论时：如果有任何评论的 parentId 指向它（有子评论）或 replyToId 指向它（被@），则软删除（isDeleted=true），显示灰色斜体"[该评论已删除]"占位（隐藏内容、作者、点赞、编辑删除等所有信息和操作，不暴露作者尤其匿名作者）；两者都没有时才物理删除
- 占位下方的子评论继续正常显示
- **重要**：主评论和子评论的渲染块都要加 isDeleted 判断（论坛之前的 bug 就是子回复渲染漏了这个判断，这次文档评论两处都要加）

### 5. @可点击跳转
- 子评论的"@某人"可点击，点击平滑滚动+高亮到被@的评论
- 给每条评论容器加 id（如 comment-{id}）和 scroll-mt 适配
- 被@的评论已软删除（占位）时，跳转到占位卡片即可

### 6. 全兼容现有机制
子评论必须和主评论一样支持：
- **匿名**：可匿名发表，匿名显示"匿名用户X"（沿用文档内帖内编号，编号在整个文档评论范围一致），@匿名对象显示"匿名用户X"不暴露
- **点赞**：复用 DocCommentLike，可点赞/取消/给自己点
- **编辑/删除**：作者改自己的、管理员删任何的，服务端校验，删除二次确认
- **头像**：显示作者头像，匿名用匿名头像
- **通知**：B回复了A的评论 → 通知A（匿名回复显示"匿名用户"不暴露）；自己回复自己不通知

## 技术要求

- DocComment 加 parentId/replyToId/isDeleted，生成 migration
- 尽量复用论坛楼中楼的组件和逻辑（内联回复框、@标签、软删除判断、@跳转高亮、匿名编号），保持论坛和文档评论体验一致
- 所有写操作走 server action，服务端严格校验权限
- 匿名严格：公开页不暴露任何匿名者身份，@对象也不暴露
- 不要改坏现有的文档评论、点赞、编辑删除、通知功能（回归）
- 出错给中文提示，不要 500
- 改完跑 npm run prisma:generate、npm run lint、npm run build，三项通过

## 验收要点（自测）

- 能回复某条评论，子评论挂主评论下，只缩进1层，显示@
- 子评论支持匿名（@匿名不泄露）、点赞、编辑删除
- 删被@过/有子评论的评论 → 变"[该评论已删除]"占位，子评论保留；**主评论和子评论占位都要正确显示**（重点测子评论的占位，论坛之前漏过）
- @可点击跳转高亮
- B回复A的评论 → A收到通知；匿名不泄露
- 现有文档评论/点赞/编辑删除/通知没被改坏
- 手机端缩进不挤压

## 本次不做

- 无限层级（只1层）
- 通知的删除/清空（另一批）
